# Investor Data Room — House of Mastery

A living folder of artifacts a technical or commercial reviewer can use to perform diligence on the House of Mastery diagnostic instruments without needing live access to source.

## Index

| Artifact | What it shows | Status |
|---|---|---|
| `01-architecture.md` | System architecture and data-flow diagram | Drafted |
| `02-security-posture.md` | Security controls, dependencies, threat model summary | Drafted |
| `03-accessibility-statement.md` | WCAG 2.2 conformance statement and VPAT-style summary | Drafted |
| `04-methodology.md` | Construct base, scoring, retake design, validation plan | Drafted |
| `05-changelog.md` | Mirror of repository CHANGELOG | Linked |
| `06-roadmap.md` | 30 / 60 / 90 / 180 day product roadmap | Drafted |
| `07-unit-economics.md` | Cohort cost, conversion funnel template, LTV worksheet | Skeleton |
| `08-quality-scores.md` | Code quality score, test coverage, Lighthouse scores | Skeleton |

## How to use

1. Reviewers should start with `01-architecture.md` for the system overview
2. Then `02-security-posture.md` for the threat model
3. Then `04-methodology.md` for the diagnostic-instrument credibility
4. Other artifacts are reference

Material is updated each release.
